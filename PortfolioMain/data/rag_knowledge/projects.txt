Project Name:
MyPortfolio – AWS EC2 Deployment with CI/CD, Nginx, and Firebase Integration

Year:
2025

Description:
A production-ready personal portfolio website with an admin dashboard, deployed on
AWS using a two-tier architecture. The system runs on an Ubuntu-based EC2 instance
and serves two Flask applications (Portfolio and Admin) through Gunicorn, with Nginx
acting as a reverse proxy. Firebase is used for authentication and realtime database
functionality. CI/CD automation is implemented using GitHub Actions to deploy updates
and restart services automatically on each code push.

Key Features:
- End-to-end CI/CD pipeline using GitHub Actions with automated EC2 deployments
- Nginx reverse proxy configuration with HTTPS via Let’s Encrypt and Certbot
- Gunicorn for production-grade WSGI execution of Flask applications
- Firebase authentication and realtime database for secure content management
- systemd services for application auto-start and process monitoring
- Custom VPC networking with routing through an Internet Gateway

Technologies Used in This Project:
- Flask
- Firebase Realtime Database
- AWS EC2 (Ubuntu)
- Nginx
- Gunicorn
- GitHub Actions
- systemd
- Certbot (Let’s Encrypt)
- Python

Scope Note:
Only the technologies listed above were directly used in the portfolio project.
Other technologies mentioned in the portfolio represent broader experience.


Project Name:
AWS 3-Tier Customer Information System

Year:
2025

Description:
A production-grade AWS-based three-tier architecture designed with separate
application, database, and networking layers, focusing on high availability,
security, and scalability.

Key Features:
- High availability architecture using Application Load Balancer and Auto Scaling
- VPC design with public and private subnets for network isolation
- Secure RDS configuration with restricted access via IAM and subnet rules
- Bastion host for controlled administrative access
- DNS configuration using Route 53 with SSL certificates managed through ACM
- Monitoring and alerting using SNS, EventBridge, and CloudWatch

Technologies Used:
- AWS EC2
- AWS RDS
- AWS VPC
- Application Load Balancer
- Auto Scaling
- AWS IAM
- Route 53
- AWS Certificate Manager (ACM)
- SNS
- EventBridge
- CloudWatch

Impact:
Demonstrated the ability to design and deploy secure, highly available cloud
architectures aligned with real-world enterprise requirements.

Year:
2025
